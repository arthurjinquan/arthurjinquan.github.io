工厂模式

介绍

简单工厂模式、工厂方法模式、抽象工厂模式，都是属于创建型设计模式。比较容易弄混淆。

工厂模式将类的创建和类的使用分离出来，当 Class A 想调用 Class B ，那么A只是调用B的方法，而至于B的实例化，就交给工厂类。

那又有人说，也可以把这些创建过程的代码放到类的构造函数里，同样可以降低重复率，而且构造函数本身的作用也是初始化对象。针对这个观点，我们可以对比下工厂模式相较于构造函数的优点：

优点：

  1. 静态工厂方法有名字而构造函数没有，因为工厂方法可以有多个，通过名字来区别各个方法，但构造函数名字只能有一个，只能通过参数来区别，所以使用静态工厂方法更明了。
  2. 静态工厂方法支持条件性实例化，就是说你创建对象时，有时需要添加一些条件判断是否应该创建，如果满足条件则返回一个实例，不满足则返回NULL，比如单例模式。构造函数时做不到这样的，构造函数的功能需要保持单一，只为了构造而存在，而静态工厂方法可以很简单的做到。
  3. 方法能够返回返回类型的子类型

工厂模式最核心的知识点是：将对象的创建和使用做分离。

参考：https://www.cnblogs.com/chinaxieshuai/p/12557538.html

简单静态工厂模式

简单工厂，首先创建phone的抽象类：

public abstract class Phone {
    public abstract void call();
}

实现小米手机和华为手机类：

public class MiPhone extends Phone {
    @Override
    public void call() {
        System.out.println("mi phone");
    }
}
public class HuaWeiPhone extends Phone {
    @Override
    public void call(){
        System.out.println("huawei phone");
    }
}

再创建phone的构建工厂类，根据传入的类型创建不同类型的手机

public class PhoneFactory {
    public static MiPhone createMiPhone(){
        return new MiPhone();
    }
    public static HuaWeiPhone createHuaWeiPhone(){
        return new HuaWeiPhone();
    }
    public static Phone createPhone(String type){
        if ("Mi".equals(type)){
            return new MiPhone();
        }else{
            return new HuaWeiPhone();
        }
    }
}

工厂方法模式

一样的先定义一个phone抽象类：

public abstract class Phone {
    public abstract void call();
}

创建一个手机的工厂接口，里面有个createPhone方法，其他类型手机都要继承这个接口

public interface PhoneFactory {
    Phone createPhone();
}

创建小米手机类。继承phone，实现call方法

public class MiPhone extends Phone {
    @Override
    public void call() {
        System.out.println("mi phone");
    }
}

创建小米手机工厂

public class MiPhoneFactory implements PhoneFactory {
    @Override
    public Phone createPhone(){
        return new MiPhone();
    }
}

华为也类似：

public class HuaWeiPhone extends Phone {
    @Override
    public void call(){
        System.out.println("huawei phone");
    }
}
public class HuaWeiPhoneFactory implements PhoneFactory {
    @Override
    public Phone createPhone(){
        return new HuaWeiPhone();
    }
}

抽象工厂模式

定义：为创建一组相关或相互依赖的对象提供一个接口，而且无须指定它们的具体类

还是用创建手机的方式来实现抽象工厂，先创建一个phone的抽象类，里面不仅有call方法，还有手机的屏幕类型

public abstract class Phone {
    public abstract void call();

    public abstract void screenType();
}

再创建一个手机构建工厂的父接口

public interface PhoneFactory {
    Phone createMiPhone();

    Phone createHuaWeiPhone();
}

首先所有小米手机和华为手机都必须实现通话功能，重写call方法（注意是抽象类）

public abstract class MiPhone extends Phone {
    @Override
    public void call() {
        System.out.println("mi phone");
    }
}
public abstract class HuaWeiPhone extends Phone {
    @Override
    public void call(){
        System.out.println("huawei phone");
    }
}

根据手机的屏幕类型分为小米type1屏手机：

public class Type1ScreenMiPhone extends MiPhone {
    @Override
    public void screenType(){
        System.out.println("type1 screen Mi phone");
    }
}

根据手机的屏幕类型分为华为type1屏手机：

public class Type1ScreenHuaWeiPhone extends HuaWeiPhone {

    @Override
    public void screenType() {
        System.out.println("type1 screen huawei phone");
    }
}

根据手机的屏幕类型分为小米type2屏手机：

public class Type2ScreenMiPhone extends MiPhone {
    @Override
    public void screenType(){
        System.out.println("type2 screen Mi phone");
    }
}

根据手机的屏幕类型分为华为type2屏手机：

public class Type2ScreenHuaWeiPhone extends HuaWeiPhone {

    @Override
    public void screenType() {
        System.out.println("type2 screen huawei phone");
    }
}

针对不同的屏幕类型手机，抽象出根据屏幕类型的构建工厂，有type1屏手机工厂和type2屏手机工厂：

public class Type1ScreenPhoneFactory implements PhoneFactory {
    @Override
    public Phone createMiPhone() {
        return new Type1ScreenMiPhone();
    }

    @Override
    public Phone createHuaWeiPhone() {
        return new Type1ScreenHuaWeiPhone();
    }
}
public class Type2ScreenPhoneFactory implements PhoneFactory {
    @Override
    public Phone createMiPhone() {
        return new Type2ScreenMiPhone();
    }

    @Override
    public Phone createHuaWeiPhone() {
        return new Type2ScreenHuaWeiPhone();
    }
}

再写一个测试类测试一下结果：

public class FactoryTest {
    public static void main(String[] args) {
        Phone miPhone = new Type1ScreenPhoneFactory().createMiPhone();
        System.out.println("miPhone的手机类型为：");
        miPhone.screenType();

        Phone huaWeiPhone = new Type2ScreenPhoneFactory().createHuaWeiPhone();
        System.out.println("HuaweiPhone的手机类型为：");
        huaWeiPhone.screenType();
    }
}

output:
miPhone的手机类型为：
type1 screen Mi phone
HuaweiPhone的手机类型为：
type2 screen huawei phone

抽象工厂模式的使用场景：一个对象族（或是一组没有任何关系的对象）都有相同的约束，则可以使用抽象工厂模式。通过工厂类，只要知道工厂类是谁，我就能创建出一个需要的对象

优缺点

缺点 ：

扩展产品族困难。比如在phone中类型增加一个带手写笔类型的手机，那么每个已经实现的手机类就都需要实现这个方法。这严重违反了开闭原则。

优点：

增加等级简单。如果在折叠屏手机下增加一个双折叠屏和三折叠屏的手机这就比较简单，只需要在折叠屏手机构建工厂下面修改就行。

总结

实际上，一般开发过程中，我们使用简单工厂模式比较多，抽象工厂模式的话需要业务比较大的情况下才会用到。
